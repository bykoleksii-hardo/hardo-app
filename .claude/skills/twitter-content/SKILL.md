# X / Twitter Content for Hardo App

Produce sharp, fast, on-brand content for X. **Always apply `hardo-brand`** for
voice, audience, pillars, and guardrails. X rewards brevity, strong opinions, and
specificity. Useful-first; sell softly.

## Before writing — quick brief (ask if missing)

1. **Goal:** reach / engagement / profile clicks / signups / community?
2. **Pillar:** reps / specific feedback / demystify / confidence / outcomes?
3. **Format:** single tweet / thread / quote-tweet / reply?
4. **Audience angle:** Target student / Switcher / Insider-multiplier (FinTwit)?

## The hook is ~70% of the work

On X the first line (and first ~5 words) decides everything — the feed is faster
and more ruthless than LinkedIn. Patterns for this niche:

- **Contrarian/strong opinion:** "Your DCF doesn't need to be perfect. Banks are testing something else."
- **Specific pain:** "The 'walk me through a DCF' freeze is real. Here's the fix."
- **Number/list:** "7 IB technicals that quietly tank candidates:"
- **Curiosity gap:** "Most candidates fail IB interviews for the same boring reason."
- **Mini-story:** "I bombed 3 IB interviews before I figured out the one thing."

Avoid throat-clearing, hashtags-in-the-hook, and corporate phrasing.

## Single tweets

- **Under 280 chars** (don't rely on Premium long-tweets for core content).
- One idea, one punch. Strong opinion or one specific, correct insight.
- Line breaks for rhythm; 0-1 emoji max (finance audience = keep it credible).
- No links in high-reach tweets — X suppresses outbound links; drop the link in a
  **reply** ("Free mock → in reply").
- 0-2 hashtags max (X is not hashtag-driven; usually 0). `$`-cashtags only if truly relevant.

### Tweet templates
- **One-liner truth:** a sharp, quotable claim tied to a pillar.
- **Tip tweet:** "Quick IB tip: when they ask X, structure it as A → B → C."
- **Mini-frame:** "Equity value vs enterprise value in one tweet: …"
- **Engagement Q:** "What's the IB technical that scared you most?" (drives replies)

## Threads (the authority + reach format)

Structure:
1. **Tweet 1 = hook + promise** ("7 technicals that tank candidates. A thread 🧵").
   The hook tweet must stand alone and earn the click.
2. **Tweets 2-N = one idea each**, concrete and correct. Short. Each tweet should be
   independently screenshot-worthy.
3. **Penultimate = recap / reframe.**
4. **Final = CTA:** follow for more + soft Hardo mention ("I built Hardo so you can
   actually *practice* these — free mock in reply").

Rules:
- 5-12 tweets. Tight. Cut filler.
- Technical accuracy is **mandatory** — a wrong DCF/LBO explanation gets ratio'd in
  FinTwit and kills credibility (see brand guardrails).
- Number the tweets (1/, 2/ …) for skimmability.
- Put the link/CTA in the **last** tweet, not the hook.

## Quote-tweets & replies (underrated growth content)

- **Quote-tweet** a relevant FinTwit/recruiter take with a sharp, additive angle
  (agree-and-extend or respectfully-disagree). Don't be a pile-on.
- **Reply hooks:** thoughtful replies to big finance accounts are a top-of-funnel
  growth lever — see `twitter-growth`. Keep them genuinely useful, not pitchy.

## "Build in public" content

FinTwit loves builders. Share Hardo's journey: what you're shipping, a metric
(only if real), a lesson, a behind-the-scenes of the question bank. This builds an
audience that roots for the product. Never fabricate numbers — use `[real metric]`.

## Output format

For a tweet request, deliver:
1. **3 hook/tweet options.**
2. The chosen **final tweet** (≤280 chars) or **full thread** (numbered).
3. **Reply text** with CTA + `[Hardo link]` + UTM suggestion.
4. One line: **why it works** (pillar + hook type).

Never invent metrics, quotes, or outcomes — bracket and flag placeholders.
