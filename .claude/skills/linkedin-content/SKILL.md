# LinkedIn Content for Hardo App

Produce scroll-stopping, on-brand LinkedIn content. **Always apply `hardo-brand`**
for voice, audience, pillars, and guardrails. Default to *useful-first* content
that earns trust; sell softly.

## Before writing — quick brief (ask if missing)

1. **Goal:** awareness / engagement / saves / signups / community?
2. **Pillar:** which messaging pillar (reps, specific feedback, demystify,
   confidence, outcomes)?
3. **Format:** text post / carousel / video script?
4. **Audience angle:** Target student / Switcher / Insider-multiplier?
5. **CTA:** which one (rotate — see brand file)?

If the user gives a loose prompt, infer sensible defaults and state them.

## Hook rules (the first 2 lines are everything)

LinkedIn truncates after ~2 lines ("...see more"). The hook must create a curiosity
or stakes gap. Patterns that work for this niche:

- **Contrarian:** "Reading WSO for 6 hours won't get you the offer. Here's what does."
- **Specific pain:** "You freeze the second they say 'walk me through a DCF.' Let's fix that."
- **Number/list:** "7 technical questions that quietly kill IB candidates."
- **Story/stakes:** "I bombed my first 3 IB interviews. Then I changed one thing."
- **Insider truth:** "Banks don't care if your DCF is perfect. They care about this."

Avoid: "I'm excited to share…", "In today's fast-paced world…", weak throat-clearing.

## Post structures

### A) Value/list post (best for saves + reach)
```
[Hook — 1-2 lines, curiosity/stakes]

[1-line setup of the promise]

[3-7 punchy points, each on its own line, scannable]
• Point — short, specific, finance-real
• Point …

[Reframe or insight that ties it together]

[CTA — soft or medium]
```

### B) Story post (best for trust + comments)
Hook → tension (the struggle) → turning point → lesson → bridge to Hardo (light)
→ CTA. Keep it human; the "before-after confidence" arc resonates hard here.

### C) "Teach one concept" post (authority)
Pick ONE technical concept (e.g. equity vs enterprise value, why we unlever beta,
the LBO "value creation" bridge). Explain it clearly and correctly in 120-180
words. This is the highest-credibility format — **technical accuracy is mandatory**
(see brand guardrails). End with: "Could you explain this out loud, fast, under
pressure? That's the real test."

## Carousels (document posts) — high saves & dwell time

8-10 slides. Rules:
- **Slide 1 = cover:** big hook, minimal words, promise the payoff.
- **Slides 2-9:** one idea per slide, big text, lots of whitespace, consistent
  layout. (Use the `design`/`ui-ux-pro-max` skills for visual specs and palette.)
- **Last slide = CTA:** save / follow / run a mock on Hardo.
- Topic templates: "10 DCF questions & how to answer", "The behavioral questions
  that trip everyone up", "Technical cheat-sheet: valuation in 9 slides",
  "Day-in-the-life of prepping with Hardo".
- Provide slide-by-slide copy + a one-line visual direction per slide.

## Short video / Reels scripts

Format: HOOK (0-3s) → value (3-25s, 1-3 fast points) → CTA (last 3s).
Give: on-screen text per beat, voiceover line, and a B-roll/visual note. Keep
total ~20-40s. Niche hooks: "POV: it's 30 seconds into your IB technical…".

## Hashtags (LinkedIn: 3-5, niche > broad)

Core set to rotate from: #InvestmentBanking #IBRecruiting #FinanceCareers
#BreakingIntoWallStreet #InterviewPrep #FinanceStudents #WallStreet #CareersInFinance.
Put 3-5 relevant ones at the end. Avoid 15-hashtag spam.

## Formatting for the LinkedIn feed

- Short lines, frequent line breaks, scannable.
- Emojis sparingly (bullets/section markers OK; don't overdo it for a finance
  audience — keep it credible).
- **Link goes in the first comment**, not the body. End the post with
  "(link in comments)".
- Ideal length: 120-220 words for text posts; the hook does the heavy lifting.

## Output format

When asked for a post, deliver:
1. **3 hook options** (so the user can pick).
2. The **full post body** (formatted for LinkedIn).
3. **Hashtags** (3-5).
4. **First-comment text** (with the CTA/link placeholder).
5. A **one-line "why this works"** note tying it to a pillar.

Never invent metrics, quotes, or outcomes — use bracketed placeholders and flag them.
