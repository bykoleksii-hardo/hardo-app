## Axis 1 — Technical

**Strengths.** Typecheck gate in CI (`tsc --noEmit`), strict TS enabled, clean module layout, server-only secrets.

**Backlog (P2/P3):** package-lock hygiene; migrate `next lint` → ESLint CLI (Next 16 deprecation); no test suite; `console.*` calls not all routed through the structured logger; missing request timeouts on some outbound fetches; README / `.env.example` drift; some migrations not committed to repo.

## Axis 2 — Security

**Strengths.** RLS on all tables (owner-scoped via `auth.uid()`); secrets encrypted in Cloudflare; most HTTP security headers present (HSTS, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy) via CF Transform Rules.

**Backlog:** **P2 — no Content-Security-Policy header.** Zod validation missing on some routes; CSRF posture; SSRF / prompt-injection hardening on AI inputs.

## Axis 3 — Data / Privacy

**Strengths.** RLS owner-scoped everywhere; no open `using(true)` policies; user IDs taken from session not request body; backups encrypted at rest (Supabase).

**Backlog:**
- **P2** — No self-serve account deletion or data export flow (GDPR art. 17/20). Currently handled by email only.
- **P2** — Logger does not redact PII (uses `JSON.stringify` on context).
- **P3** — 10 empty/near-empty `_backup_*` tables; DROP script provided to owner to run.
- **P3** — Profile update route lacks Zod schema / field-length limits.

## Axis 4 — Billing (LemonSqueezy)

**Both P1 issues fixed during audit.**
- **P1 (fixed, PR #15)** — Subscription period now enforced server-side (paid tier requires active status + non-expired period).
- **P1 (fixed, DB direct)** — Out-of-order webhook protection: `apply_lemonsqueezy_event` now uses `greatest(existing, incoming)` on `current_period_end` so a delayed/stale event cannot shorten an active subscription. Applied to staging (smoke-tested) then prod; admin entitlement verified intact. **Migration file not yet committed — tracked as a follow-up (see audit/p2 issue); the fix is live in prod.**

**Backlog (P2):** full event-id dedup not possible with current schema; refund / payment_failed events ignored; webhook HMAC compare not constant-time; schema drift — a 2026_01 migration targets `public.users` while runtime uses `user_entitlements`.

## Axis 5 — UX / Visual

**Fixed (PR #16).** Responsive app shell: header padding `px-5 md:px-12`, profile tabs horizontal-scroll on mobile, display name hidden on small screens. Content grids were already responsive.

**Backlog (P3):** dual score-scale inconsistency; no drop-off handling; "Manage Plan" vs "coming soon" wording.

## Axis 6 — Content / Legal copy

**Fixed (PR #17).** Terms §3 price corrected $12 → $14.99 (matches landing/Pricing). Privacy §6 + Terms §7 deletion wording made honest ("by emailing hello@hardo.app") since no self-serve flow exists. Free-tier limit (1 Intern interview per account, hard-blocked thereafter) **verified correctly implemented** in `get_user_quota_status` + `interview/start` — no change needed.

**Backlog:** build self-serve export/delete (ties to Axis 3 P2); enforce real age-gate (16+, currently only stated); Knowledge Hub content sparse (owner task, issue #3).

## Axis 7 — Operations

**Strengths.** Secrets encrypted in Cloudflare (no plaintext leakage); Worker observability logs enabled (100% sampling); cron-worker `hardo-cron` runs every 15 min (downgrade expired subs + publish due articles), checks response `.ok` and logs each task outcome; Sentry server-side intentionally disabled with documented upstream incompatibility (opennextjs/cloudflare), errors captured via `onRequestError` → `logger.error`.

**Backlog:**
- **P2** — CI does not run lint and has no tests; workflow stops at a TODO after typecheck.
- **P2** — cron `scheduled()` has no try/catch; a network throw in `fetch` aborts both tasks for that tick without a structured error log (re-runs in 15 min mitigates).
- **P2** — no CSP header (shared with Axis 2).
- **P3** — observability Traces/Exports not configured; open-next has no incremental cache (R2/KV) for ISR.

## Axis 8 — Legal / Corporate

No code fix possible — findings require owner-supplied legal information or infra decisions.
- **P1 (owner input)** — No legal entity / data-controller identification in Terms or Privacy (no entity name, country, registered address; only `hello@hardo.app`). Required for GDPR if serving EU/UK users.
- **P1 (owner input)** — No governing law / dispute-resolution clause in Terms.
- **P2** — GDPR framework incomplete in Privacy: no explicit "data controller", no legal bases for processing, no right-to-complain-to-supervisory-authority, retention periods only stated for server logs (30 days).
- **P2** — LemonSqueezy as Merchant of Record not disclosed (Privacy says only "our payment processor"; Terms silent on VAT/tax/MoR).
- **P2** — Production domain `hardo.app` not connected to the app: email domain is verified in Resend (EU/Ireland), but the web app is only reachable at the `workers.dev` dev subdomain while legal copy and branding already reference `hardo.app`.
